# Roadmap: current state → first working version

Scout-grounded, phased plan from the current Claude-Code-based v2 motor system to a
first working version that embodies the thesis end-to-end. Each phase carries a
**runnable verification gate** that must pass before the next begins.

> **Status: proposed.** Generated by a read-only scout+synthesis pass; ground-truth
> the file/line refs before relying on them, and pin the baseline (P1) before
> trusting "no regression".

## Definition of the first working version

You launch `orca` (TUI), converse with an L3 primary agent in a **non-blocking
braid**, and it reifies work into a durable SQLite action/edge **circuit that
contains a loop** (a back-edge with a close/escape condition). The circuit's agent
actions execute end-to-end against a model you **choose per task** (≥1 Anthropic and
≥1 OpenAI-family model in a single build) through Orca's **own agent loop**, with
**no `claude` binary on PATH** and **no `@anthropic-ai/claude-agent-sdk`** in either
`package.json`. Every graph mutation (L3 agent and L2 supervisor) is **validated
before commit and rolled back** if it would corrupt the circuit; a **global
circuit-breaker** caps total cost and graph size. The build runs under `orca serve`;
the TUI attaches over REST+SSE and **detaches on quit while the build keeps
running**; the build is **resumable** because agent session/context is persisted to
SQLite.

**Explicitly NOT in the working version** (post-1.0): concurrent/parallel
scheduling, per-action scope, computed-goto routing, web multiplexer.

## Critical path

**P1 → P2 → P3 is the spine and the critical path.** The ModelProvider keystone (P1)
unblocks Orca-owned Layer B + direct Anthropic inference (P2) — the single longest,
highest-risk pole. P3 deletes the SDK and adds a second provider to prove
model-agnosticism. **P4 (governance) is independent of provider work and can be built
in parallel on a second track, but must merge before P5**, because P5 turns the L3
agent's tools into graph mutations and would otherwise amplify the live
unvalidated-mutation hazard (`supervisor.ts:180-187` silent catch). P5 unblocks the
TUI (P6), which is otherwise low-coupling since the daemon/SSE substrate already
exists.

## Phases

### P1 — ModelProvider seam (pure refactor, no behavior change) · risk: low
**Goal:** introduce the Layer-A `ModelProvider` interface + shared neutral types and
route the existing runtime through it, keeping `invoke()`/`invokeSimple()` signatures
byte-identical. Safe vertical slice; pins the regression baseline for every later
phase.

**Builds:**
- New `packages/server/src/models/`: `types.ts` (`ModelMessage`, `ToolSchema`, `ToolCall`, `ModelDelta`, `Usage`), `provider.ts` (the `ModelProvider` interface + capability flags), `registry.ts` (model-id prefix → provider + price + capabilities).
- `ClaudeCodeProvider` adapter wrapping today's SDK path so `engine/invoke.ts:97-242` drives the provider instead of calling `query()` inline; `findClaudeExecutable()` stays but moves behind the adapter.
- Refactor `harness/loop.ts` `callApi` so both `action.type='agent-api'` and `'agent'` resolve a provider from the registry (no new behavior).
- **Baseline pin:** `scripts/baseline.json` recording the exact current pass count so "no regression" is mechanically checkable despite the known-flaky tests.

**Verification — gate:** `models/*.test.ts` pass; full suite ≥ pinned baseline (no NEW failures vs the recorded flaky set); `invoke()` signature snapshot unchanged; no production code touched outside `engine/invoke.ts`, `harness/loop.ts`, `models/`.
```
cd packages/server && SKIP_LIVE=1 bun test src/models/
cd packages/server && SKIP_LIVE=1 bun test 2>&1 | tail -5
cd packages/server && bun test src/engine/invoke.test.ts
```

### P2 — Orca-owned Layer B + direct Anthropic inference (no claude binary) · risk: high
**Goal:** promote the agent loop to a real Orca-owned Layer B with streaming, durable
sessions, and lossless cost; make `AnthropicProvider` a streaming Messages-API
adapter and the **default** path. The longest pole and the classic bug source.

**Builds:**
- `engine/agent-loop.ts` (Layer B): turn loop calling `provider.stream()`; built-in tool registry keyed by `TOOLSETS` reusing `harness/tools.ts`; scope gate via `checkToolUse`; `maxTurns`; structured-output finalization (explicit "which turn is final" logic); `InvokeEvent` emission tagged with source action id.
- Streaming `AnthropicProvider`: SSE Messages API with `input_json_delta` tool-arg accumulation by content-block index, parsed at `content_block_stop`; `Usage` carries the raw `cache_creation`/`cache_read` split.
- Cost from raw `Usage` × registry price table (replaces SDK `total_cost_usd`).
- Durable sessions: new SQLite table so conversation state survives restart; loop persists+resumes by `sessionId` (replaces SDK `resume`). Resolves the spec's open session surface.
- Wire `v2/action-runner.ts` `runAgentAction` to default to `agent-loop.ts`; keep `ClaudeCodeProvider` behind an opt-in flag as a one-phase fallback.

**Verification — gate:** tool-arg accumulation + cost-from-usage + session-resume tests pass; a live ≥2-turn tool-using run (reads a file, returns validated structured output, cost>0) completes with `claude` **not resolvable on PATH**; scope violation still denies; suite ≥ baseline.
```
cd packages/server && SKIP_LIVE=1 bun test src/engine/agent-loop.test.ts src/models/
cd packages/server && SKIP_LIVE=1 bun test src/v2/ 2>&1 | tail -5
# Strip only the dir that holds `claude` from PATH (keeping bun); prove claude is
# unresolvable, then run the live loop. If ANTHROPIC_API_KEY is set this hits the
# real API; otherwise it drives the identical loop over a hermetic Anthropic-wire
# mock so the end-to-end path still runs with real evidence.
bash -c 'CD=$(command -v claude 2>/dev/null); if [ -n "$CD" ]; then export PATH=$(echo "$PATH" | tr ":" "\n" | grep -v "^$(dirname "$CD")$" | paste -sd ":" -); fi; command -v claude && echo CLAUDE_PRESENT_FAIL || (cd packages/server && SKIP_LIVE=0 bun test src/engine/live.test.ts)'
```

### P3 — Second provider (OpenAI) + cut the Claude Code dependency · risk: high
**Goal:** force the abstraction honest with a real non-Anthropic adapter, then delete
the SDK and executable resolution so model-agnosticism is literally true.

**Builds:**
- `models/openai.ts`: message-shape translation (Anthropic blocks ↔ OpenAI `role:'tool'` keyed by `tool_call_id`), tool-arg fragment accumulation by tool index, strict `response_format: json_schema`, `stream_options:{include_usage:true}`; `Usage` tolerates `cacheWrite=0`.
- Registry: provider-prefixed ids; `resolveModelId` replaced by registry lookup; `MODEL_ALIASES` retired.
- Remove `@anthropic-ai/claude-agent-sdk` from both `package.json`s; delete `findClaudeExecutable()` and the `preset:'claude_code'` system-prompt append; drop `settingSources`.
- A two-provider fixture build (one action pinned `anthropic/*`, one `openai/*`).

**Verification — gate:** both providers pass a shared live structured-output/tool contract; the two-provider fixture build completes with combined cost; `grep` finds **zero** references to `claude-agent-sdk`/`findClaudeExecutable` in src and both manifests; suite ≥ baseline.
```
cd packages/server && SKIP_LIVE=1 bun test src/models/openai.test.ts
bash -c '! grep -rn "claude-agent-sdk\|findClaudeExecutable" packages/server/src package.json packages/server/package.json'
cd packages/server && SKIP_LIVE=0 bun test src/models/two-provider.live.test.ts
```

### P4 — Governance: validated mutation (DRC) + global circuit-breaker · risk: medium
*(Parallelizable with P2/P3; must merge before P5.)*
**Goal:** close the unvalidated-mutation hazard before the conversational agent
multiplies mutation volume.

**Builds:**
- Strengthen `validateGraph` (`graph-ops.ts:136-196`): distinguish a **legal loop** (back-edge with close/escape condition) from an illegal unbounded cycle; add reachability + size caps.
- Replace `supervisor.ts:180-187` silent catch with a DRC path: validate the prospective post-delta state, apply inside the existing `BEGIN/COMMIT/ROLLBACK` transaction, and on failure ROLLBACK + record an `invalid_mutation` history event. Distinguish validation errors (reject) from execution errors (retry).
- A reusable `applyValidatedDelta()` used by **both** supervisor and (P5) the L3 agent — one governed chokepoint.
- Global circuit-breaker in `v2/executor.ts`: cap total cost + graph size; on breach, halt and escalate to the primary agent/human, emitting an SSE `unhandled_failure`.

**Verification — gate:** 10+ adversarial mutation scenarios all roll back with no silent swallow; a legal self-loop-with-escape is **accepted**; the breaker halts+escalates on cost and on size breach; suite ≥ baseline.
```
cd packages/server && SKIP_LIVE=1 bun test src/v2/supervisor-validation.test.ts src/v2/graph-ops.test.ts
cd packages/server && SKIP_LIVE=1 bun test src/v2/circuit-breaker.test.ts
cd packages/server && SKIP_LIVE=1 bun test src/v2/ 2>&1 | tail -5
```

### P5 — L3 primary agent: converse → reify a looping circuit (loopcraft) · risk: high
**Goal:** build the conversational L3 agent whose **tools are graph mutations**, so a
chat request reifies work into the circuit and can construct a feedback loop.

**Builds:**
- L3 agent runtime on `engine/agent-loop.ts` with a graph-mutation toolset (`add_action`/`add_edge`/`update`/`remove`) that calls `applyValidatedDelta()` (P4) — it cannot commit an invalid or unbounded circuit.
- Conversational entry: POST a user message → L3 turn → emits **braid messages** (source-tagged) **and** graph-edit deltas in the same step.
- Loopcraft prompting/templates so the agent builds a build→test→route-back loop with an escape condition + a max-iteration cap.
- Server route + SSE event for L3 messages so chat is non-blocking (POST returns immediately; narration arrives as SSE), reusing `broadcast()`.

**Verification — gate:** a conversation produces a circuit whose `validateGraph` passes AND contains a back-edge with an escape condition; the executor runs the loop, it iterates >1 and **terminates via the escape** (not the max-iteration breaker); an invalid mutation the agent requests is rejected by P4 governance.
```
cd packages/server && SKIP_LIVE=1 bun test src/v2/l3-agent.test.ts
cd packages/server && SKIP_LIVE=1 bun test src/v2/loop-circuit.test.ts
cd packages/server && SKIP_LIVE=0 bun test src/v2/l3-agent.live.test.ts
```

### P6 — Conversational TUI (braid + live circuit), attach/detach · risk: medium
**Goal:** ship the default front-end — a non-blocking braid + a live list-mode circuit
pane that attaches to `orca serve` over REST+SSE and detaches on quit while the build
keeps running. The last mile to the working version.

**Builds:**
- `packages/tui`: Ink two-pane shell compiled into the bun binary; raw-ANSI/list circuit canvas; top bar (counts/cost/burn/elapsed) + bottom bar (keybindings).
- SSE consumer that coalesces events and caps redraws (~15fps), virtualized rows.
- Conversational control: input box POSTs to the L3 endpoint (P5); agent mutations render as circuit-edit cards in the braid and update the circuit pane within one redraw.
- Lifecycle: `orca` auto-starts a daemon if none; `q` detaches and the build provably keeps running; node-select detail pane + per-node `p`/`x`/`r` steer.

**Verification — gate:** TUI acceptance #1+#2 (read-only shell streams braid+circuit and detaches without stopping the build; a chat message routes to L3 and its mutation renders as a card + circuit update within one redraw); end-to-end: launch → converse → looping circuit builds and runs on a chosen model with no claude binary, build survives detach.
```
cd packages/tui && bun test
cd packages/server && SKIP_LIVE=1 bun test src/v2/tui-detach.test.ts
bash -c 'cd packages/server && bun run build && command -v claude && echo FAIL_CLAUDE_PRESENT || echo no-claude-ok'
```

## Open risks

1. **P2 streaming tool-arg accumulation** is the named classic bug source (buffer by content-block/tool index; parse at `content_block_stop`; OpenAI puts the name only in the first fragment). A subtle off-by-index bug corrupts tool inputs silently — gate with the canned-chunk unit harness.
2. **Structured-output finalization** — can't force a synthetic output-tool while real tools are in play; Layer B must decide the "final answer" turn. Anthropic forced-output-tool vs OpenAI strict `json_schema` is a real cross-provider parity risk.
3. **Session persistence is an OPEN spec surface** — `harness/loop.ts` has none; P2 builds it net-new, and the durable schema (what to store; agent re-read context vs braid) is load-bearing for "resumable" and P6.
4. **The baseline is muddy** — suite reports ~778 pass / ~22 skip / ~34 fail / 7 errors with ~19–34 known-flaky. "No regression" is only meaningful once **P1 pins an explicit baseline** and later phases diff against the *named* flaky set, not a raw count.
5. **`validateGraph` is partial today** (no cycle/reachability/size checks) — P4 must separate a legal loop from an illegal unbounded cycle, or it will reject the very loops the working version needs (P5) or fail to stop runaways.
6. **P6 TUI** (Ink + custom ANSI canvas, SSE coalescing/virtualization, `NO_COLOR` degradation) has zero current tests and no Ink dep yet; detach-keeps-running must be proven by integration test, not assumed.
7. **Cost-table drift** — Orca now owns price math across providers; a stale price silently corrupts the breaker (P4) and TUI burn-rate. Needs one registry source of truth + a sane-cost test per model id.
8. **Cut-over ordering in P3** — removing the SDK while any default path still imports it breaks the build; the grep gate must cover all of src + both manifests, and the P2 `ClaudeCodeProvider` fallback must be fully retired first.
